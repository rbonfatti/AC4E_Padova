# Padova Workshop Exercises

Use this folder during the live workshop. It follows `SCHEDULE.md` and points to
the exact files participants edit, inspect, or copy.

| Time | Block | Exercise file | Participant output | Verification check |
| --- | --- | --- | --- | --- |
| 00:00-00:15 | Orientation | `00_orientation.md` | Tool lane chosen and repo map understood | Explain which lane and folder you will use. |
| 00:15-00:45 | Foundations | `01_foundations.md` | First scoped read-only prompt | Agent reports scope, evidence, and human-owned claims. |
| 00:45-01:20 | Card-Krueger scaffold | `02_card_krueger_scaffold.md` | Draft brief or data/design memo row | Required Card-Krueger fields are visible. |
| 01:30-02:15 | Harness pieces | `03_harness_pieces.md` | One harness component installed or mapped | Lane README path and copied component are recorded. |
| 02:15-03:00 | Orchestration | `04_orchestration.md` | Small issue or goal prompt | Allowed files, forbidden files, acceptance, and review are present. |
| 03:00-03:45 | Research sprint | `05_research_sprint.md` | One bounded artifact changed or drafted | Diff or draft has a verification note. |
| 03:45-04:30 | Verification | `06_verification.md` | Review, replication, or Playwright evidence log | Green/yellow/red result is recorded. |
| 04:30-05:00 | Transfer | `07_transfer.md` | 7-day adoption plan | First task is smaller than one day and has a check. |

General boundaries:

- Do not put private data, raw restricted data, `.env`, secrets, or confidential
  manuscripts into any agent prompt.
- Use the synthetic Card-Krueger teaching data only for workflow practice.
- The human owns identification, interpretation, citations, and final claims.
- If a task grows beyond one file set, open an issue instead of expanding the
  prompt.
